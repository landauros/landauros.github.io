从 1965 年的分时系统到 2026 年的 LLM API：计算范式的轮回

通过交互式 CRT 终端体验分时系统，再理解 LLM API 为什么像 2026 年的主机时代。

引子：为什么我想把主机时代重新做一遍#

大多数人理解 LLM API 的方式，是把它看成一个“更聪明的搜索框”，或者一个“挂在云端的聊天机器人”。这当然没错，但它会遮住一个更重要的问题：

我们今天与大模型交互的方式，其实非常像 1960 年代人们使用大型主机的方式。

当时，一台机器太昂贵，个人不可能拥有。于是用户拿着终端，通过线路连到远端主机，把任务提交过去，等待调度器分配计算时间，再取回结果。今天你在浏览器里输入 prompt，通过网络把请求送到 GPU 集群，由远端推理系统排队、调度、计费，再把输出返回给你。硬件变了，界面变了，范式几乎没变。

这个栏目想做的，不只是“讲历史”，而是让你先亲手体验一次这种范式，再回来理解它。

1965：一台主机，多个终端#

分时系统诞生的背景很简单：算力极贵，用户很多。

如果每个研究员都想独占一台机器，那么大多数时间机器都在等待人类输入；这在经济上完全不可接受。于是系统设计者开始反过来思考：

  1. 让多个人同时连接同一台主机
  2. 让系统把 CPU 时间切成小片轮流分配
  3. 让每个用户感觉自己“像是独占了一台机器”

这就是分时系统最重要的心理魔术。你看到的是一个属于自己的 shell，背后其实是调度器在不停切换上下文。

在上面的演示里，who、ps、top、run hello.f 这些命令，都是为了把这个事实直接暴露给你看。

权限不是附属品，而是共享的前提#

多用户共享主机后，第二个问题马上出现：既然大家在同一台机器上，边界在哪里？

于是文件权限系统出现了。你可以看到 /home 下有多个用户目录，但你不能直接读 prof_chen 的私有研究记录；与此同时，shared_data.csv 又是可读的，因为它被显式设计成了共享文件。

这很像今天的大模型 API：

  • 你和别人共用同一套 GPU
  • 但你看不到别人的上下文
  • 你只能访问系统允许你访问的资源

在主机时代，这叫文件权限与账户隔离。今天，它更常被包装成 API Key、租户隔离、会话上下文与权限域。

时间片调度，后来变成了推理调度#

如果说权限解决的是“边界”问题，那么时间片解决的是“公平”问题。

在分时系统里，调度器要决定：

  • 哪个用户现在拿到 CPU
  • 一个作业能运行多久
  • 什么时候被更高优先级任务抢占

在今天的大模型基础设施里，名字换了，但问题几乎一样：

  • 哪个请求先进入队列
  • 哪个请求被分配到哪张 GPU
  • 在高负载下，如何保证整体吞吐与单用户延迟

所以当你在演示里看到：

  • run hello.f 触发时间片变化
  • future 之后 queue 展示推理排队
  • quota 展示 token 额度与消耗

其实你看到的不是两种系统，而是同一套资源治理逻辑的两个历史阶段。

Prompt，可能就是今天的“程序”#

这个类比里最有趣的一点，是“程序”的位置变了。

1965 年，你给主机的输入更像一段 FORTRAN 程序。2026 年，你给模型的输入更像一段自然语言 prompt。两者的形式差别很大，但它们有一个共同点：

它们都在告诉远端共享计算资源：你想让它替你做什么。

所以我们可以粗暴但有启发地说：

  • FORTRAN 程序是那个时代提交给主机的任务描述
  • Prompt 是这个时代提交给模型的任务描述

这当然不是严格的一一对应，但它足够抓住真正重要的结构。

为什么历史会押韵#

每当一种新的计算形态刚出现，而且资源极其昂贵时，社会都会自然而然回到“集中式共享”的模式。

主机时代如此，云 GPU 时代也是如此。因为在硬件成本还没有低到人人都能本地拥有之前，共享是唯一经济可行的方案。

后来，随着成本下降，计算能力才会慢慢下放：

  • 主机走向个人电脑
  • 也许云端 LLM 会逐渐走向本地模型

如果这个判断成立，那么今天的 LLM API 时代，不一定是终局。它更像一次历史回声中的中间阶段。

如何使用这个栏目#

如果你想顺着它最初设计的教学节奏体验，可以按这个顺序试：

  1. ls /home
  2. cat /home/prof_chen/research.txt
  3. cat /home/prof_chen/shared_data.csv
  4. cat hello.f
  5. run hello.f
  6. future
  7. send "什么是注意力机制"
  8. compare

这一圈走完，通常就能非常直观地感受到：LLM API 不是突然从天上掉下来的新物种，它更像主机时代共享计算逻辑的一次现代复现。

小结#

如果只用一句话总结这篇文章和上面的互动演示，那就是：

你今天调用大模型的方式，本质上仍然是在向一台你并不拥有、但被你暂时借用的远端主机提交任务。

1965 年，人们把它叫做 timesharing。2026 年，我们把它叫做 inference API。
